# RecolorAI

เว็บเติมสีภาพขาวดำด้วย AI สำหรับโฮสต์บน GitHub Pages หรือเว็บสเตติกทั่วไป

## คุณสมบัติ

- อัปโหลดภาพ PNG / JPG จากเครื่องผู้ใช้
- ประมวลผลด้วยโมเดล AI บนเบราว์เซอร์โดยตรง
- ไม่ต้องใช้เซิร์ฟเวอร์สำหรับการประมวลผลภาพ
- มีปุ่มดาวน์โหลดภาพที่ได้
- รองรับการใช้งานต่อยอดด้วยโครงสร้างโฟลเดอร์ที่เป็นระเบียบ

## โครงสร้างโฟลเดอร์

- `index.html` - หน้าเว็บหลัก
- `css/styles.css` - รูปแบบสวยงามและ responsive
- `js/app.js` - โค้ดจัดการการอัปโหลดภาพและเรียกใช้ ONNX Runtime
- `.gitignore` - ข้ามไฟล์ที่ไม่จำเป็น
- `README.md` - คู่มือสั้น ๆ

## วิธีใช้งาน

1. เปิดไฟล์ `index.html` ด้วยเว็บเซิร์ฟเวอร์หรือ GitHub Pages
2. อัปโหลดภาพ JPG / PNG จากเครื่อง
3. รอให้โมเดลโหลดและประมวลผล
4. กดปุ่มดาวน์โหลดเพื่อบันทึกภาพสี

## การโฮสต์บน GitHub Pages

1. สร้าง repository ใหม่บน GitHub
2. คัดลอกไฟล์ทั้งหมดไปยัง repository
3. ตรวจสอบให้มีไฟล์ `package.json` และโฟลเดอร์ `.github/workflows/deploy.yml`
4. ตั้งค่า GitHub Pages ให้ใช้งานจาก `main` branch
5. เมื่อ `main` มีการ push GitHub Actions จะทำงานอัตโนมัติและ deploy เว็บไซต์ให้
6. เข้าใช้งานผ่าน URL ของ GitHub Pages ที่ GitHub แจ้งหลัง deploy เสร็จ

> หมายเหตุ: โมเดล ONNX โหลดจาก CDN ภายนอก เพื่อไม่ให้ repository หนักเกินไปและลดขนาดการเก็บไฟล์

## GitHub Actions Deployment

ไฟล์ workflow อยู่ที่ `.github/workflows/deploy.yml`

- จะทำงานเมื่อ push ไปยัง branch `main`
- ติดตั้ง Node.js และ dependencies
- สร้าง artifact จากไฟล์ใน repo
- deploy ไปยัง GitHub Pages โดยอัตโนมัติ

ถ้าต้องการตรวจสอบสถานะการ deploy ให้ไปที่ `Actions` ในหน้า repository ของ GitHub แล้วเลือก workflow `Deploy to GitHub Pages`.

## วิธีทดสอบท้องถิ่น

1. รันคำสั่ง `npm install` เพื่อโหลด `http-server`
2. รัน `npm run start`
3. เปิดเบราว์เซอร์ที่ `http://localhost:8080`

## การรักษาความปลอดภัย

- ใช้ `Content-Security-Policy` จำกัดแหล่งที่มาของสคริปต์และไฟล์
- จำกัดประเภทไฟล์ให้เฉพาะ `image/png` และ `image/jpeg`
- จำกัดขนาดไฟล์ไม่เกิน 5MB
- ไม่ส่งภาพขึ้นเซิร์ฟเวอร์ ภาพประมวลผลบนเครื่องผู้ใช้เอง
- มีการจัดเก็บโค้ดแยกไฟล์แบบ modular เพื่อให้พัฒนาต่อได้ง่าย

## ขยายต่อในอนาคต

- เปลี่ยนโมเดล ONNX เป็นเวอร์ชันใหม่หรือเวอร์ชันที่โหลดจากโฮสต์ของคุณเอง
- เพิ่มการจัดการรูปแบบภาพเพิ่มเติม เช่น WebP หรือ BMP
- เพิ่มฟีเจอร์ปรับขนาดภาพก่อนประมวลผล
- เพิ่มระบบบันทึกเวอร์ชันภาพหรือภาพก่อน-หลัง
